# 최유라쇼 편성 스케줄러

롯데홈쇼핑 [최유라쇼] 테마프로그램 편성 관리 웹앱.
MD팀이 상품을 입찰하고, PD가 드래그앤드롭으로 편성하고, 배정·변경이력까지 추적합니다.

## 실행 방법

빌드/설치가 필요 없습니다.

- **간단히**: `index.html` 더블클릭 (인터넷 연결 필요 — React·Tailwind·폰트를 CDN에서 로드)
- **권장(로컬 서버)**: 폴더에서
  ```
  python3 -m http.server 4178
  ```
  → 브라우저에서 `http://localhost:4178`

데이터는 브라우저 **localStorage**에 저장됩니다(같은 브라우저에서 새로고침해도 유지).

## 화면

- **PD 편성표**: 좌측 입찰 풀에서 카드를 시간대로 드래그하여 편성. 카드 간 이동, 시간 분할(⊟), 시간대/편성일 추가·삭제, 카드의 ✎로 PD·쇼호스트·스튜디오·분량 배정.
- **MD 입찰보드**: 팀 선택 → 시간대별 `+ 입찰`로 상품명·내용·이슈·구성·가격·마진·분량·중소/특약 입력. 중복 입찰 가능.
- **변경 이력**: 우상단 버튼. 상품별 편성 이동 횟수 요약 + 전체 변경 로그(편성/이동/제외/입찰/시간편집)를 from→to로 추적.

## 기본 편성 (2026년 7월 시드)

- 목요일: `20:45~21:45`(60분), `21:45~22:50`(65분)
- 토요일: `08:20~09:20`, `09:20~10:20`, `10:20~10:35`(15분)
- 특별편성: 시간대 추가/분할, 임의 요일(편성일) 추가 지원

## 파일 구조

```
index.html     진입점 (CDN 로드 + 마운트)
dataStore.js   데이터 계층 (도메인 모델 + 변이 + 변경이력 자동기록)
app.js         UI (편성표 / 입찰보드 / 이력 — htm + React, 무빌드)
```

## 2단계 — Supabase로 멀티유저/서버저장 전환

`dataStore.js`는 **데이터 계층 인터페이스**로 추상화되어 있습니다.
UI(`app.js`)는 `store.getState() / subscribe / 변이함수`만 사용하므로,
`LocalStore`를 동일 인터페이스의 `SupabaseStore`로 교체하면 UI 수정 없이 전환됩니다.

전환 시 매핑:
- 테이블: `teams`, `days`, `slots`, `bids`, `placements`, `change_log`
- 실시간: Supabase Realtime 구독 → `subscribe` 콜백에 연결 (PD 편성 변경이 다른 접속자에 즉시 반영)
- 인증/권한: Supabase Auth + RLS (MD는 자기 팀 입찰, PD는 편성)
- 변경이력: DB 트리거로 `change_log` 자동 적재 (현재 `log()` 로직과 1:1 대응)

`window.createDataStore()`에서 `SUPABASE_URL` 존재 시 `SupabaseStore`를 반환하도록 분기 지점이 이미 준비돼 있습니다.
